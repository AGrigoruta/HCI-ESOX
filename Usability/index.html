Usability Test Plan - EsoX

Usability Test Plan Section

Test Plan for: EsoX

Persona Type Tested: Primary/Secondary/Served

Context: A meeting room from the office
Duration: 10 minutes
Overall Objectives
  • Assess the effectiveness of Esox by having users perform the main flows
  • Identify obstacles to completing a certain task or using a certain component;
  • Evaluate consistency and standards;
  • Evaluate accessibility features
  • Assess the user experience while using EsoX
Scenario

The user gets familiarized with a new application for educational purposes, EsoX, an application for learning Esoteric Programming Languages in a user friendly way, having all of these tutorials centralized in one place. They either want to learn something new and exciting, or they are just curious about what EsoX is all about. They will start navigating the application.

Tasks
  • Task 1: Log in the application, save an intermediate course and join a guild. - Estimated time: 3 minutes;
  • Task 2: Start a Piet tutorial and run a program. - Estimated time: 6 minutes;
  • Task 3: Start a Befunge tutorial, have a 500ms delay on, and run a program. - Estimated time: 5 minutes;
Post-test Questions
  • How easy was the application to navigate through?
  • How good were the tutorials explained?
  • What would you improve to the overall functionality of the application?
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".

Test Results for: EsoX

Success Criteria

A successful design has been achieved when:

  • 1 out of 3 users managed to complete a task in both Befunge and Piet tutorials.
  • 2 out of 3 users managed to save a course and join a guild
  • 3 out of 3 users managed to log in the application
  • 3 out of 3 users managed to filter the courses

Summary

  • Two users went to the Community tab and used the search functionality
  • A user didn't notice the example in Piet and navigated through the tutorial for 2 minutes without direction
  • A user played with Befunge's delay function and enjoyed the runtime animation
  • A user tried to break the application by running all the programs in a tutorial
  • Two users used the import picture functionality in Piet
