Abstract

This document is is meant to surmise the thought process that took place while implementing the requirements for the HCI homework.

Introduction

EsoX (Esoteric Language User Experience), is a project that is meant to imagine various user experiences regarding multiple ways of interaction with an esoteric programming language, plus activities of (e.g., learning, programming, translating into a general purpose language, research,...) and related tools like interpreters, compilers, development environments, notable applications, and others. The developed application will also recommend other (esoteric) programming languages according to user preferences, language categories (paradigm, usability, computational class, other features), pragmatic usages, etc.

The requirements for the first part of the project presentation are as fallows:

  • Minimum 3 ideas/concepts regarding the project design (including the sketches, wireframes, storyboards and, optionally, prototypes), plus the intermediary stages of the project development. Also, the team should provide the documentation – as Scholarly HTML – regarding interaction design patterns used in the wireframes/prototypes.
  • Minimum 3 real examples concerning the QOC method. These examples could also give the motivation for choosing a certain idea/concept.
  • The 4 mandatory HTML5 persona documents, for various personas interacting to the system.
  • The project identity and the most significant details regarding the branding process.

Given the short but wide description there is a lot of room to maneuver so first thing we should narrow down the requirmeents to something that can be managed.

To summarize the content of the image: Esox mission is to offer new ways of interaction in order to learn programming or to program. This will be done by creating new tools or enhancing existing ones and by exploring diffrent "cultures" of giving introduction to machines.

Q1: How do we gratify the user’s progress and motivate him to learn more?

As a solution, we could implement both achievements/badges in the user profile and also progress bars on every Esoteric Language. The incentives could be a great idea to motivate and reward, but for digital incentives we depend on third party providers (like having Amazon/any other e-store wallet credit) and for physical incentives we have to manage user addresses and deliveries, and issues could arise from the delivery process itself so big exclamation mark in the feasibility criteria.

Q2: How will the user navigate through the application?

We could either implement the breadcrumbs and menus to navigate back and forward which will be rather static on the dashboard, so the user will always know his navigation location, and have a screen which changes the content according to the clicks (easier to navigate with the mouse), or we could implement tabs and cards and have a more game-like dashboard (easier to navigate with the game controller). For the speech navigation, this also enables the use of the app for people without an available mouse/keyboard, so they could use it with their voice and perhaps integrate it with their smart speakers (Alexa, Google Home, etc.)

Q3: How will the application form a community and create some level of competitivity between users?

In order to motivate and instigate comptetitiveness between users, we could organise monthly/quarterly challenges and competitions, with a form of reward for the winners. If we want them to collaborate together, we could implement learning groups. This way they could learn faster from one another, while also competing to be the best group/ guild from the application, like in a MMORPG game. The leaderboards are also in the equation, and they would display the most hard-working users in the app, for the user to see and strive to take their place.

User Guide

Use Case 1

The first use case will present how to navigate through the website and start working on a course.First you will be greeted on the homepage where you can see and sort all the courses available.

Before selecting a course though you should login. You can do this by clicking the "Login" button on the top right. You have the option to login through Facebook, Google, Git or with a website specific account.

After selecting a course you can opt to participate in it. The page will show you your time spent on the course, the achievements you unlocked and your friends that also participated in this course. By clicking the green "play" button you will be redirected to the lesson page that also includes the coding language interpretor, with examples.

Use Case 2

The second use case will present how a client can join a guild. As in case 1 the user will have to login to access the guild features. Upon entering the "Community" tab you will be presented with multiple available guilds on the left side. Each guild's page will show you your friends that are part of it, the guild's achievements and competitions that they took part in or will take part in. To join a guild simply press the blue "+" button under the guilds name.You can quit the guild the same way. You can only be part of a single guild at a time.
